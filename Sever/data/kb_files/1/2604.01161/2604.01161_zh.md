# 推理偏移：上下文如何悄然缩短大语言模型的推理过程

格列布·罗季奥诺夫∗ Yandex

# 摘要

展现出测试时缩放（test-time scaling）行为（例如扩展的推理轨迹与自我验证）的大语言模型（LLMs），在复杂、长程推理任务中已展现出卓越性能。然而，此类推理行为的鲁棒性仍缺乏深入探究。为此，我们在三种场景下对多个推理模型开展了系统性评估：（1）问题附加了冗长且无关的上下文；（2）多轮对话设置中各轮任务相互独立；（3）问题作为复杂任务中的一个子任务呈现。我们观察到一个有趣现象：相较于问题单独呈现时所生成的推理轨迹，同一问题在不同上下文条件下所触发的推理轨迹显著缩短（最多达 $50\%$）。更细粒度的分析表明，这种压缩与自我验证及不确定性管理等行为（如反复核查）的减少密切相关。尽管该行为偏移在简单问题上并未损害模型性能，但在更具挑战性的任务中可能造成性能下降。我们希望本研究能进一步引起学界对推理模型鲁棒性，以及大语言模型与基于LLM的智能体之上下文管理问题的关注。

# 1 引言

近期，测试时缩放作为一种有前景的策略崭露头角，其通过在推理阶段分配更多计算资源，使模型能够根据输入复杂度动态调整自身行为，而无需重新训练 [OpenAI et al., 2024, DeepSeek-AI et al., 2025]。测试时缩放的一个典型范例是思维链（Chain-of-Thought, CoT）[Kojima et al., 2022, Wei et al., 2022]。借助强化学习（RL）及其他后训练技术的最新进展，大语言模型现已具备一种“思考模式”，使其能在给出用户查询答案前生成较长的推理轨迹。尤为重要的是，当前推理型大语言模型的诸多进步，往往与其对自身解题过程进行自我反思的能力密切相关——这一点体现于其CoT中出现的高层级模式，例如回溯（backtracking）、不确定性管理（uncertainty management）与自我验证（self-verification）[Gandhi et al., 2025, Venhoff et al., 2025]。这些模式通常伴随着推理轨迹长度的增加。

与此同时，近年来长上下文语言模型迅速兴起，其上下文窗口已扩展至数十万乃至数百万词元（tokens）[Dubey et al., 2024, Anthropic, 2024, Google DeepMind, 2025b]。这些进展为测试时缩放提供了充足空间，使得模型可在单一上下文中完成探索、反思与总结，从而支持复杂推理 [Liu et al., 2025a]、多阶段“证明者–验证者”流水线（prover-verifier pipelines）[Shao et al., 2025]，以及更为精细的智能体工作流（agent workflows）[Team et al., 2025]。然而，多项研究已揭示长上下文所伴随的显著局限性，例如：从长上下文中检索相关信息的能力下降 [Kamradt, 2023]、上下文内学习（in-context learning）能力减弱 [Li et al., 2024]、在多轮对话中难以从错误假设中恢复 [Laban et al., 2026]，以及在长输入上执行多步推理的能力受限 [Ling et al.,

2025]。此外，Du 等人 [2025] 还证实，仅输入长度本身即足以损害大语言模型的性能，且该效应独立于检索质量，并在完全不存在干扰信息的情况下依然存在。

随着多种测试时策略及面向长期任务的智能体不断涌现，一些自然问题随之产生：“上下文长度与内容如何影响模型的推理能力？”尤其值得追问的是：“若模型面对一个孤立的子问题，但其上下文中混杂了无关数据，其求解方式是否会与该问题单独呈现时一致？”

本文研究了一种令人惊讶的现象：我们观察到，模型在不同上下文条件下求解同一问题时，其行为分布发生了显著偏移。我们重点考察了若干简单干扰条件对推理质量与性能的影响：（1）问题附加了冗长且无关的上下文；（2）多轮对话设置中各轮任务相互独立；（3）问题作为复杂任务中的一个子任务呈现。特别地，我们发现：当问题处于非孤立上下文条件下时，推理模型所生成的推理词元数量显著减少（最多减少至原来的 $1/2$）。对推理链的分析表明，这种压缩现象与自我验证及不确定性管理等行为（如反复核查）的减少密切相关。尽管推理轨迹的缩减可能在较简单问题上缓解过度推理，且不牺牲准确率，但在更具挑战性的任务中却会导致性能下降。

我们希望本研究能进一步引起学界对推理模型鲁棒性，以及大语言模型与基于LLM的智能体之上下文管理问题的关注。

# 2 背景

思维链推理（Chain-of-Thought reasoning）。测试时缩放带来了一场范式变革，使长程思维链推理成为可能，并激发了复杂的推理行为，从而显著提升模型在竞赛级数学与编程任务中的表现。推动这一变革的核心技术是大规模强化学习（large-scale RL），其可诱导出诸如自我验证与迭代精炼（iterative refinement）等复杂推理行为 [OpenAI et al., 2024, DeepSeek-AI et al., 2025, Shao et al., 2024, Yu et al., 2025, Kumar et al., 2024]。后续研究亦探索了其他实现测试时缩放与强推理性能的路径 [Muennighoff et al., 2025, Ye et al., 2025]。

额外的推理预算为根据输入问题的复杂度动态调整推理努力程度带来了新的挑战 [Yang 等, 2025a]。Su 等 [2025] 分析了推理长度与答案正确性之间的关系，发现大语言模型（LLMs）在较简单的问题上倾向于过度推理，在较难的问题上则推理不足，表明模型可能无法据此校准其推理长度。Aggarwal 等 [2025] 提出了一种统一基准，可联合评估 LLMs 中的过度推理与推理不足现象。

为分析 LLMs 如何在其冗长的思维链（CoT）轨迹中应对高难度推理问题，Venhoff 等 [2025] 在单条推理轨迹内定义了若干不同的推理功能，并利用一个 LLM 判定器对每句话按其功能角色进行分类。Bogdan 等 [2025] 借鉴该框架，提出通过识别引导推理路径走向的关键步骤，将句子归入不同类别并量化其因果影响，来分析长推理轨迹；该研究特别强调了规划类语句或不确定性管理类语句的重要性。Muennighoff 等 [2025] 同样证明：当模型试图终止推理时，若人为干预其推理轨迹以强制其进行自我反思，则可能提升模型性能。后续研究进一步探讨了准确性与推理轨迹高层特征（如长度、复核比例等）之间的关系 [Wu 等, 2025；Hassid 等, 2025；Feng 等, 2025]。

长上下文与上下文管理。近年来，模型支持的上下文长度实现了显著增长 [Dubey 等, 2024；Anthropic, 2024；Google DeepMind, 2025b]。尽管上下文长度的扩展令人瞩目，但这些模型所宣称支持的上下文长度与其实际能有效处理的上下文长度之间仍存在显著差距 [Liu 等, 2025b]。常见局限通常体现在基于检索的评估任务 [Kamradt, 2023] 以及多跳追踪与聚合任务 [Hsieh 等, 2024] 中。然而，Du 等 [2025] 表明，即使在完美完成检索的合成设定下（即模型被强制仅关注相关 token），长上下文仍可能导致性能下降。Laban 等 [2026] 则通过分片模拟（sharded simulation）——即用一组较小的指令共同传达原始指令的全部信息——在多轮对话设定中观察到显著的性能下降。在此设定下，模型往往过度依赖其在前几轮中做出的错误假设。

为克服长上下文的固有局限并进一步释放测试时计算资源的扩展潜力，研究者提出了多种方法，包括上下文压缩、迭代式摘要生成以及外部记忆模块 [Anthropic, 2025；Liu 等, 2025a；Yan 等, 2025；Tekparmak 和 Ömer Kaya, 2025]。另一条并行的研究路线则充分利用可被拆分为相互独立子问题的任务特性，通过递归式自调用（recursive self-calls）将子问题交由模型自身分别处理；该策略不仅可提升效率，还能自然地维持紧凑的上下文表征 [Yang 等, 2025b；Jin 等, 2025；Ning 等, 2024；Zheng 等, 2025]。

# 3 实验

# 3.1 实验设置

本节实验旨在回答一个关键问题：当一个模型在充斥着无关上下文的环境中求解某个孤立子问题时，其表现是否仍能与在无干扰的孤立环境下相当？该问题的提出基于两项观察：第一，复杂的推理任务通常可被分解为若干彼此独立、无需全局上下文即可求解的子任务；第二，长期运行的智能体日益频繁地工作于宽泛而通用的上下文中，而这类上下文不可避免地包含大量与特定用户查询或子任务无关的细节信息。

为评估模型在不同上下文条件下对同一问题的推理能力，我们比较以下四种实验设定：

- • 基线设定（Baseline）：模型仅接收一条用户消息，其中包含一个标准提示下的问题；  
- • 子任务设定（Subtask）：模型仅接收一条用户消息，其中包含两个需独立求解的问题；  
- • 长输入设定（Long input）：模型仅接收一条用户消息，其中先是一大段无关数据，随后才是标准提示下的问题；  
- • 多轮对话设定（Multi-turn）：模型接收一段多轮对话历史，其中每条用户消息均使用基线提示要求模型求解一个新问题；我们仅评估第二轮的响应表现。

# 4 分析

第 3 节中的实验表明，在不同上下文条件下，同一问题所生成的推理 token 数量存在显著差异。本节将通过检查 Qwen3-32B 在 Baseline 与 Long input 设置下针对 MATH500 问题所生成的推理轨迹（reasoning traces），来深入分析这些轨迹之间的差异。

首先，我们人工检查了这些推理轨迹，以寻找上下文可能影响推理过程的任何证据——我们并未发现模型因查询内容而产生困惑或未能理解任务的迹象。具体而言，在考察 Qwen3-32B 在“长输入”提示下生成的推理轨迹时，我们仅发现其对无关提示部分有极短暂的提及；若干示例见附录 E。这些提及均为简短且带有否定意味的确认：模型注意到前缀内容，将其标记为“无关”，并立即转向后续推理。我们未发现任何证据表明模型在语义层面实际参与或受到所插入内容的影响。

接着，我们比较 Baseline 轨迹是否因在候选答案得出后展现出更强的验证模式，而导致其抵达最终答案所需时间更长或整体长度更长。为此，我们在每条轨迹中识别出首个候选答案出现的位置。有趣的是，尽管平均推理长度存在显著差异（3824 个 token vs. 2741 个 token），但首个候选答案位置的平均值与中位数却几乎一致（平均分别为 925 与 939 个 token，两种设置下的中位数均接近 400 个 token）。需指出的是，由于该模型在本基准测试上的性能差异较小（96.5 vs. 95.0），我们未评估首个候选答案的正确性。未来工作可考虑在更具挑战性的问题上进一步分析该效应。

为深入理解推理长度显著差异的本质，我们采用 [Venhoff et al., 2025] 提出的框架，对长推理轨迹开展句子级分析。我们将每条轨迹中的每个句子归类为以下类别之一：问题设定（problem setup）、计划生成（plan generation）、事实检索（fact retrieval）、主动计算（active computation）、结果整合（result consolidation）、不确定性管理（uncertainty management）以及最终答案输出（final answer emission）。我们进一步考察这些标签在推理轨迹内部的转移矩阵（transition matrices）。图 4 展示了 Baseline 与 Long input 设置下转移热力图的差异。值得注意的是，该热力图中绝对值最大的一项为“最终答案输出 → 推理轨迹结束”的转移概率：Baseline 下为 $57\%$，而 Long input 下升至 $68\%$。这可能揭示了一种显著的行为差异：一旦最终答案被陈述，Long input 的轨迹更倾向于就此终止；而 Baseline 轨迹则更可能启动额外的自我核查步骤。

然而，这些变化可能源于推理轨迹自身的内容特征，而非上下文条件本身。为验证这一点，我们开展了一项重采样实验（resampling experiment）：我们取 Long input 条件下生成的推理轨迹，从其推理部分末尾移除少量 token（最后 50 个 token），然后在三种不同上下文条件下重新采样相同数量的 token：Baseline、Long input，以及一种类似 Long input 但提示更短的条件（仅插入 128 个 token）。对于重采样所得的轨迹，我们比较其中“已完成推理”（即包含推理结束或自我验证 token）的轨迹比例。结果见表 3。Long input 条件下完成推理的轨迹比例为 $46\%$，而 Baseline 条件下仅为 $21\%$，该差异亦伴随……

# 5 讨论

在本工作论文中，我们发现不同的上下文条件可能影响推理型大语言模型（LLM）解决同一问题的方式。具体而言，我们证实了诸如不确定性管理与自我验证等高层次行为模式的分布具有脆弱性，容易被提示（prompt）中无关的上下文所抑制。尽管对于较简单的问题，这种抑制可能减少过度推理，但此类行为转变却会降低模型在更具挑战性的任务上的表现。

局限性与未来工作  
我们希望强调本文当前版本存在的若干重要局限性。首先，我们所采用的上下文条件相对简单且为人工构造；在未来工作中，亟需在更真实的情境（包括具身智能体（agentic）场景）中演示并分析上述推理转变现象——即所谓“野外环境”（in the wild）下的研究。其次，第 4 节中的分析仅聚焦于单一模型；其他模型推理长度较短的原因可能另有他因。此外，当前版本仅涵盖数学问题，尚未涉及其他对推理能力要求较高的领域，例如编程或自然科学。最后，本文目前既未涵盖也未提出任何缓解该推理转变的方法，例如提示调优（prompt tuning）、更稳定的训练技术，或上下文管理方法（包括基于递归自调用的方法 [Yang et al., 2025b; Jin et al., 2025; Ning et al., 2024; Zheng et al., 2025; Zhang et al., 2025]）。
